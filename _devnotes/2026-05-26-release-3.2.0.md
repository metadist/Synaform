## Engine changes (`SynaformController.php`)

### Fix #1 — checkbox SDT post-pass tolerates inline content between rPr and `<w:t>`

`convertCheckboxMarkersInPart()` accepts the v4 hhff shape

```
<w:r><w:rPr>…</w:rPr><w:tab/><w:t>{{checkb.moving.yes}} Ja</w:t></w:r>
```

The pre-fix regex required `<w:t>` to follow `</w:rPr>` directly — a
`<w:tab/>` between them broke the match and every Bereitschaft checkbox
ended up as literal `[[SYNCB|on|☒|☐]]` text. The new regex captures the
inline middle (`<w:tab/>`, `<w:br/>`, `<w:sym/>`, `<w:noBreakHyphen/>`,
`<w:cr/>`, etc.) and preserves it on the surviving "before" run so the
visual layout — e.g. the leading tab that separates the checkbox glyph
from its label — is kept intact.

Regression coverage: new `tests/phase-e2-checkbox-sdt-with-tab.php`
exercises three real-world inline shapes (tab, break, tab + symbol);
the existing `tests/phase-e-checkbox-sdt.php` was updated for the new
8-group capture pattern. Both pass.

End-to-end verification on the live v4 hhff template:

```
SYNCB literal count:  0
<w:sdt> count:         6
</w:sdt> count:        6
<w14:checkbox> count:  6
```

(3 Bereitschaften × 2 paired glyphs = 6 SDTs.)

### Fix #2 — `cleanTemplateMacros` promotes bare `<w:tr>` rows

PhpWord's `TemplateProcessor::findRowStart()` searches BACKWARD for
`<w:tr ` (with the trailing space — i.e. a row carrying at least one
attribute) before falling back to bare `<w:tr>`. When a customer's
template uses bare `<w:tr>` for the row hosting `{{stations.*.N}}`
placeholders and the previous table on the page closes with an
attributed `<w:tr w:rsidR=…>`, findRowStart matches the previous table's
row, the slice spans both tables, and `cloneRow` produces a corrupted
document where the previous table's last row is duplicated INTO the
Werdegang table.

We hit this end-to-end on the first v5 NeedleHaystack attempt —
"sonstige Kenntnisse / `{{other_skills}}`" leaked into the cloned
stations table. Fix is a deterministic pre-pass that injects a synthetic
`w:rsidR="00000000"` attribute onto every bare `<w:tr>` during
`cleanTemplateMacros`. The transform is idempotent and schema-valid,
and pre-existing attributed rows are left untouched (so designer-set
`w:rsidR` revision marks survive intact).

Regression coverage: new `tests/phase-c2-clonerow-bare-tr.php`.

### New — Template Doctor

`lintTemplate(string $docxPath)` runs at upload time and on demand, returns
`{ ok, summary, findings[] }` with five rules:

| Code | Severity | Catches |
|---|---|---|
| `MULTIPLE_PLACEHOLDERS_NO_SEPARATOR` | error | Two placeholders fused with no separator (`{{a}}{{b}}`) |
| `LIST_INLINE_TEXT` | warning | List-typed placeholder shares paragraph with non-empty text |
| `BARE_TR_ROW_GROUP_HOST` | info | Row-group placeholder lives in a bare `<w:tr>` (engine auto-fixes, but worth tidying) |
| `LIST_OR_ROW_IN_HEADER_FOOTER` | warning | List/row placeholder in a header or footer (engine renders body-only) |
| `PLACEHOLDER_INSIDE_SDT` | warning | `{{...}}` nested inside a content control — substitution can't reach it |

Surfaced via:

- `POST /templates` response now includes `template.lint`.
- New `GET /templates/{id}/lint` endpoint for re-running the audit on demand.
- Frontend (`index.js`):
  - errors → blocking modal listing each finding with a one-line fix,
  - warnings/infos only → blue info toast pointing the user to the modal,
  - clean templates → existing green "Saved!" toast.

i18n keys added to `en.json`, `de.json`, `es.json`, `tr.json` (parity
verified). Plugin version bumped from `3.1.0` → `3.2.0`.

## Customer templates

Two new templates in `hhff/word-files/v5/`:

- `Profil hhff DE v5.docx` (84 KB, 32 placeholders)
- `Profil NeedleHaystack DE v5.docx` (59 KB, 29 placeholders)

Both are byte-stable rebuilds from the v4 master copies via
`_devnotes/build-v5-templates.php`. The transformation only touches the
Werdegang region — every other paragraph, header, footer, image, list
and content-control is preserved verbatim from v4. Re-running the build
script overwrites the v5 outputs.

Werdegang shape after rebuild (both templates):

```
<w:tbl>
  <w:tblGrid>
    <w:gridCol w:w="2300"/>   <!-- ~4.0 cm time column -->
    <w:gridCol w:w="6900"/>   <!-- ~12.0 cm content column -->
  </w:tblGrid>
  <w:tr w:rsidR="00000001" w:rsidTr="00000001">
    <w:trPr><w:cantSplit/></w:trPr>
    <w:tc>{{stations.time.N}}</w:tc>
    <w:tc>
      <w:p>{{stations.employer.N}} (bold)</w:p>
      <w:p>{{stations.position.N}}</w:p>
      <w:p>{{stations.details.N}}</w:p>
    </w:tc>
  </w:tr>
</w:tbl>
```

Smoke-tested via `_devnotes/v4-api-smoketest.php` (which renames itself
generically — works for both v4 and v5). Verified output for v5 hhff:

```
PASS — placeholder substitution, header/footer substitution, font
preservation (Arial × 76, MS Gothic × 6) all OK; stations table renders
with two rows (one per station) carrying employer / position / details
in the right cell.
```

Verified output for v5 NeedleHaystack:

```
PASS — Helvetica-Light × 39, MS Gothic × 6, Bebas Neue × 2; both
stations now render in a single 2-row table; sonstige Kenntnisse no
longer leaks into the Werdegang.
```

## AI configuration

User 1's `BCONFIG.DEFAULTMODEL.PIC2TEXT` switched from `BID=17`
(Llama 4 Scout / Groq, 0.11 €/M in) to `BID=171`
(Gemini 2.5 Flash / Google, 0.30 €/M in). Per-image wall-clock on the
real interview scans:

| Image | Size | Strategy | Before (Llama 4 Scout / Gemini 2.5 Pro fallback) | After (Gemini 2.5 Flash) |
|---|---:|---|---:|---:|
| IMG_2116.JPG | 4.30 MB | vision_ai | 72.9 s | **12.75 s** |
| IMG_2117.JPG | 3.43 MB | vision_ai | 70.0 s | **8.06 s** |
| IMG_2118.JPG | 3.99 MB | vision_ai | 44.0 s | (~9 s expected) |
| Total (3 files) | 11.7 MB | — | 186.9 s | **~30 s** |

Average per image: **62 s → ~10 s** (≈ **6×** faster on the slowest case,
**9×** on the typical case). Quality stays usable for the form-filling
prompt that follows: candidate names, dates, project codes and checkbox
states are all read correctly.

Cost: at $0.30 / M input tokens, processing one 4 MB scan
(~3 000 input tokens) costs ~$0.001 — negligible against the 60 s of
human time saved per image.

## Documentation

- New: `docs/TEMPLATE_AUTHORING.md` — the 10 rules HHFF designers need
  to keep their Word templates from breaking. Covers placeholder
  separators, list paragraph hygiene, single-row repeating tables,
  attribute-bearing `<w:tr>`, body-only list expansion, real Word
  tables vs tabs, no nested SDTs, paired checkbox shape, autocorrect
  defragmentation, font inheritance. Pairs with the Template Doctor —
  every rule maps to a doctor finding code so the customer can resolve
  issues without our involvement.
